SPI Communications Lab 7

This was a lab for my embedded software class. It involved writing drivers for SPI communication for one of the onboard sensors. To read and write from the sensor a state machine was used. This information was then processed within the application code and caused the boards LED to turn green if it was upside down and off if it was rightside up. A message was also sent to a bluetooth module that told you which direction it was facing as well.
